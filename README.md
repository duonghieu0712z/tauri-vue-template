# Tauri Vue Template

Minimal Tauri 2 + Vue 3 + TypeScript template with Vite, Tailwind CSS, Specta bindings, backend logging, and a ready lint/format workflow.

## Stack

- Tauri 2
- Vue 3
- TypeScript
- Vite
- Tailwind CSS 4
- Specta and tauri-specta
- Oxlint, ESLint, and Oxfmt

## Setup

```bash
pnpm install
```

## Development

```bash
pnpm start
```

Use `pnpm dev` only when running the Vite frontend without the Tauri shell.

## Build

```bash
pnpm make
```

`pnpm make` runs the frontend build through Tauri's `beforeBuildCommand`, then builds the app bundle.

To build only the frontend:

```bash
pnpm build
```

## Checks

```bash
pnpm format
pnpm lint
pnpm type-check
```

To apply automatic fixes:

```bash
pnpm format:fix
pnpm lint:fix
```

## Rename Template

Use the rename script after creating a new app from this template:

```bash
pnpm rename -- --name "My App" --id "com.example.my-app"
```

By default, the package and Rust crate names are derived from the app name. To set the package name explicitly:

```bash
pnpm rename -- --name "My App" --id "com.example.my-app" --package "my-app"
```

After renaming, regenerate lockfiles with the package managers:

```bash
pnpm install
cargo check --manifest-path src-tauri/Cargo.toml
```

## Bump Version

```bash
pnpm bump 0.1.0
```

The version script updates `package.json`, `src-tauri/tauri.conf.json`, and `src-tauri/Cargo.toml`. Lockfiles are generated by their package managers.

## Generated Files

`src/generated/` is reserved for Specta output and is ignored by git. The Tauri backend exports bindings in debug builds.

## Icons

The browser favicon lives in `public/`. Tauri bundle icons live in `src-tauri/icons/`.

## Recommended IDE Setup

Use VS Code with Vue - Official, Tauri, and rust-analyzer.
